# overview

一个代理框架（harness）是由多种不同能力组合而成，使得构建长期运行的代理更加容易：

- 规划能力
- 虚拟文件系统
- 文件系统权限
- 任务委派（子代理）
- 上下文与令牌管理（上下文工程）
- 代码执行
- 人机协同
- 框架配置文件

除了这些能力，Deep Agents 还使用技能（Skills）和记忆（Memory）来提供额外的上下文和指令。

## 规划能力

该框架提供了一个 `write_todos` 工具，代理可以用它来维护一份结构化的任务清单。

**功能：**

- 用状态（`'pending'`、`'in_progress'`、`'completed'`）跟踪多个任务
- 持久保存在代理状态中
- 帮助代理组织复杂的多步骤工作
- 对长期运行的任务和规划非常有用

## 虚拟文件系统访问

该框架提供一个可配置的虚拟文件系统，可由不同的可插拔后端支持。
这些后端支持以下文件系统操作：

| 工具          | 描述                                                                                                                                                   |
| ------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `ls`          | 列出目录中的文件及元数据（大小、修改时间）                                                                                                             |
| `read_file`   | 读取带行号的文件内容，支持对大文件进行偏移/限制。还支持为非文本文件（图像、视频、音频和文档）返回多模态内容块。见下方支持的扩展名。                     |
| `write_file`  | 创建新文件                                                                                                                                             |
| `edit_file`   | 在文件中执行精确的字符串替换（支持全局替换模式）                                                                                                       |
| `glob`        | 查找匹配模式的文件（例如 `**/*.py`）                                                                                                                   |
| `grep`        | 搜索文件内容，支持多种输出模式（仅文件名、带上下文的内容或计数）                                                                                       |
| `execute`     | 在环境中运行 shell 命令（仅在沙箱后端可用）                                                                                                            |

| 类型   | 扩展名                                                                      |
| ------ | --------------------------------------------------------------------------- |
| 图像   | `.png`、`.jpg`、`.jpeg`、`.gif`、`.webp`、`.heic`、`.heif`                  |
| 视频   | `.mp4`、`.mpeg`、`.mov`、`.avi`、`.flv`、`.mpg`、`.webm`、`.wmv`、`.3gpp`   |
| 音频   | `.wav`、`.mp3`、`.aiff`、`.aac`、`.ogg`、`.flac`                            |
| 文件   | `.pdf`、`.ppt`、`.pptx`                                                     |
**running without the default filesystem tools**
要向模型隐藏以上列出的文件系统工具，请通过 `excluded_tools` 注册一个框架配置文件：

```python
from deepagents import HarnessProfile, register_harness_profile

register_harness_profile(
    "anthropic:claude-sonnet-4-6",
    HarnessProfile(
        excluded_tools=frozenset(
            {"ls", "read_file", "write_file", "edit_file", "glob", "grep"}
        ),
    ),
)
```

通过 `excluded_middleware` 移除 `FilesystemMiddleware` 本身会被有意地拒绝 —— 请使用 `excluded_tools` 仅隐藏模型可见的工具接口，而保留中间件。如需移除 `task` 工具，请参阅“在没有子代理的情况下运行”。

虚拟文件系统还被框架的其他多项能力所使用，例如技能、记忆、代码执行和上下文管理。
在构建 Deep Agents 的自定义工具和中间件时，你也可以使用该文件系统。

更多信息请参见[后端](中文/deepagent/Backends.md)。

## 文件系统权限

框架支持声明式权限规则，控制代理可以读取或写入哪些文件和目录。权限应用于上述内置文件系统工具，并按声明顺序以“最先匹配优先”的语义进行求值。

**工作原理：**

- 在创建代理时将规则列表传递给 `permissions=` 参数
- 每条规则指定 `operations`（`"read"` 或 `"write"`）、`paths`（glob 模式）和 `mode`（`"allow"` 或 `"deny"`）
- 最先匹配的规则生效。如果没有规则匹配，则允许该操作。

**为什么有用：**

- 将代理限制在特定目录中（例如 `/workspace/`）
- 保护敏感文件（例如 `.env`、凭证文件）
- 为子代理赋予比父代理更窄的访问权限

权限不适用于沙箱后端，因为沙箱后端支持通过 `execute` 工具执行任意命令。如需自定义验证逻辑，请使用后端策略钩子。

有关完整的规则结构、示例和子代理继承，请参见权限。

## 任务委派（子代理）

框架允许主代理创建临时的“子代理”来执行隔离的多步骤任务。

**为什么有用：**

- **上下文隔离** – 子代理的工作不会污染主代理的上下文
- **并行执行** – 多个子代理可以并发运行
- **专业化** – 子代理可以拥有不同的工具/配置
- **令牌效率** – 大型子任务上下文被压缩为单个结果

**工作原理：**

- 主代理拥有一个 `task` 工具
- 调用该工具时，它会创建一个全新的、带有自己上下文的代理实例
- 子代理自主执行直至完成
- 返回一份最终报告给主代理
- 可以使用默认的 `general-purpose` 子代理（默认启用）或添加自定义子代理
- 子代理是无状态的（无法发回多条消息）

**Running without subagents (no `task` tool)**

要在没有 `task` 工具的情况下运行代理，请参阅“在没有子代理的情况下运行”。不要试图通过 `excluded_middleware` 移除 `SubAgentMiddleware` —— 这会被有意地拒绝。相反，应通过框架配置文件禁用自动添加的子代理，并通过 `subagents=` 不传递任何同步子代理。异步子代理不受影响。

## 上下文管理

框架负责管理上下文，使深度代理能够在令牌限制内处理长期运行的任务，同时保留所需的信息。

**工作原理：**

- **输入上下文** — 系统提示、记忆、技能和工具提示决定了代理启动时知晓的内容
- **压缩** — 内置的卸载和摘要功能随着任务的进行，将上下文保持在窗口限制内
- **隔离** — 子代理隔离繁重的工作并仅返回结果（见任务委派）
- **长期记忆** — 通过虚拟文件系统实现跨线程的持久存储

**为什么有用：**

- 支持超出单个上下文窗口的多步骤任务
- 保持最相关的信息在范围内，无需手动裁剪
- 通过自动摘要和卸载减少令牌使用量

有关配置详情，请参见上下文工程。

## 代码执行

Deep Agents 通过两种方式支持代码执行：

- 沙箱后端暴露一个 `execute` 工具，用于在隔离环境中执行 shell 命令。
- 解释器添加一个 `eval` 工具，用于在限定了作用域的 QuickJS 运行时中运行 JavaScript。

当代理需要安装依赖项、运行测试、调用命令行界面或与操作系统文件系统交互时，可使用沙箱后端。沙箱后端实现了 `SandboxBackendProtocolV2`；当框架检测到它时，会向代理的可用工具中添加 `execute` 工具。

当代理需要一个轻量级的可编程层，用于循环、批处理、确定性数据转换或程序化工具调用时，可使用解释器。解释器不提供 shell 访问、软件包安装、文件系统及网络访问。

有关沙箱设置、提供商和文件传输 API，请参见沙箱。有关 QuickJS 运行时和程序化工具调用，请参见解释器。

## 人机协同

框架可以在指定的工具调用处暂停代理执行，以允许人工审批或修改。此功能通过 `interrupt_on` 参数选择加入。

**配置：**

- 将 `interrupt_on` 传递给 `create_deep_agent`，其中包含工具名称到中断配置的映射
- 示例：`interrupt_on={"edit_file": True}` 会在每次编辑前暂停
- 在提示时，你可以提供审批消息或修改工具输入

**为什么有用：**

- 为破坏性操作设置安全闸门
- 在昂贵的 API 调用之前进行用户验证
- 交互式调试和指导

## 技能

该框架支持技能，为你的深度代理提供专业化的工作流程和领域知识。

**工作原理：**

- 技能遵循 Agent Skills 标准
- 每个技能都是一个包含 `SKILL.md` 文件的目录，该文件包含指令和元数据
- 技能可以包含额外的脚本、参考文档、模板及其他资源
- 技能采用渐进式披露 —— 仅当代理确定它们对当前任务有用时才会加载
- 代理在启动时读取每个 `SKILL.md` 文件的前置元数据（frontmatter），然后在需要时审阅完整的技能内容

**为什么有用：**

- 通过仅在需要时加载相关技能来减少令牌使用量
- 将各种能力与额外上下文捆绑成更大的动作
- 在不使系统提示变得杂乱的情况下提供专业知识
- 实现模块化、可复用的代理能力

有关更多信息，请参见技能。

## 记忆

该框架支持持久性的记忆文件，为你的深度代理提供跨对话的额外上下文。
这些文件通常包含通用的编码风格、偏好、惯例和指南，帮助代理理解如何处理你的代码库并遵循你的偏好。

**工作原理：**

- 使用 `AGENTS.md` 文件提供持久上下文
- 记忆文件总是会被加载（不同于技能，后者采用渐进式披露）
- 在创建代理时，将一个或多个文件路径传递给 `memory` 参数
- 文件存储在代理的后端（StateBackend、StoreBackend 或 FilesystemBackend）中
- 代理可以根据你的交互、反馈和识别到的模式来更新记忆

**为什么有用：**

- 提供无需在每次对话中重新指定的持久上下文
- 用于存储用户偏好、项目指南或领域知识
- 对代理始终可用，确保行为一致

有关配置详情和示例，请参见记忆。

## 框架配置文件

每当选择了特定的提供商或模型时，该框架都可以应用一套声明式配置包（`HarnessProfile`）。配置文件在模型构建完成后调整运行时行为，无需为每个代理编写设置代码。

**工作原理：**

- 以提供商名称（`"openai"`）或 `provider:model` 组合键（`"openai:gpt-5.4"`）注册一个配置文件
- `create_deep_agent` 在解析模型时会查找并应用该配置文件
- 提供商级别和模型级别的配置文件在解析时合并

**为什么有用：**

- 将按提供商或按模型的默认设置（系统提示调整、工具覆盖、中间件）集中打包在一处
- 切换模型时，`create_deep_agent` 的调用点保持不变
- 通过入口点以插件形式发布可复用的配置文件

有关完整的字段列表、合并语义和插件打包，请参见配置文件。
