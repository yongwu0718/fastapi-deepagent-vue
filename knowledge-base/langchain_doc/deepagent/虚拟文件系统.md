我举一个非常具体的例子，把整个过程串起来。

---

## 假设你的实际文件夹结构长这样

```
F:\agent\langchain-docs\
├── index\
│   ├── readme.md
│   └── langgraph\
│       └── doc.md
└── public\
    └── notes.txt
```

---

## 情况一：root_dir 设成上一级（权限能生效）

```python
backend = FilesystemBackend(
    root_dir=r"F:\agent\langchain-docs",
    virtual_mode=True,
)
```

这时候，虚拟文件系统会这样映射：

| 虚拟路径 | 实际磁盘路径 |
|---------|------------|
| `/` | `F:\agent\langchain-docs\` |
| `/index/` | `F:\agent\langchain-docs\index\` |
| `/index/readme.md` | `F:\agent\langchain-docs\index\readme.md` |
| `/index/langgraph/doc.md` | `F:\agent\langchain-docs\index\langgraph\doc.md` |
| `/public/notes.txt` | `F:\agent\langchain-docs\public\notes.txt` |

**此时权限规则就能直接匹配虚拟路径**：

```python
permissions = [
    FilesystemPermission(
        operations=["read"],
        paths=["/index/**"],  # 匹配虚拟路径 /index/ 下的所有内容
        mode="allow",
    ),
    FilesystemPermission(
        operations=["write"],
        paths=["/**"],
        mode="deny",
    ),
]
```

- 代理调用 `ls /` 会看到 `index`、`public` 两个子目录（如果权限未阻止列出，具体看权限实现）。
- 代理调用 `read_file /index/readme.md` → 权限检查：虚拟路径 `/index/readme.md` 匹配 `/index/**`，**允许读取**。
- 代理调用 `read_file /public/notes.txt` → 不匹配任何允许规则，因为没有定义它的访问权限，默认**允许**（取决于你的权限策略）。
- 任何写入操作都会被全局拒绝，因为 `write` 的 `/**` 规则会匹配所有虚拟路径。

---

## 情况二：root_dir 直接设成 index 文件夹本身（权限失效）

```python
backend = FilesystemBackend(
    root_dir=r"F:\agent\langchain-docs\index",
    virtual_mode=True,
)
```

这时候虚拟文件系统变了：

| 虚拟路径 | 实际磁盘路径 |
|---------|------------|
| `/` | `F:\agent\langchain-docs\index\` |
| `/readme.md` | `F:\agent\langchain-docs\index\readme.md` |
| `/langgraph/doc.md` | `F:\agent\langchain-docs\index\langgraph\doc.md` |

**注意：虚拟路径里已经没有 `/index/` 这个层级了**。因为根目录本身就已经是 `index` 文件夹了。

此时你如果还用权限规则：

```python
paths=["/index/**"]  # 试图匹配虚拟路径 /index/...
```

就**永远匹配不到任何文件**，因为虚拟路径中根本不存在 `/index/` 这个目录。结果就是，你设置的权限形同虚设，所有操作都退回到默认的“允许”（除非你另有全局拒绝规则）。

---

## 一句话总结

> **权限规则的“根”就是后端的 `root_dir` 转成的虚拟根 `/`。**
>
> 设置 `root_dir` 的时候，要让它包含你想用权限控制的所有子目录。那些你想限制访问的文件夹，应当作为虚拟根下的子目录出现，这样你才能在权限规则里用 `/子目录/**` 去匹配它们。

如果你希望只暴露 `index` 文件夹内的内容给代理，但同时又要对 `index` 内部的不同子路径做精细权限控制，那 `root_dir` 就得设为 `F:\agent\langchain-docs`（即上一级），然后通过权限规则去限制只能访问 `/index/**`，并拒绝写入。
