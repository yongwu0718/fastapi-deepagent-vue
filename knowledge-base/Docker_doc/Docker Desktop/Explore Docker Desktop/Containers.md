## 探索 Docker Desktop 中的 Containers 视图

**Containers** 视图列出了所有运行中和已停止的容器及应用程序。它提供了一个简洁的界面，用于管理容器的生命周期、与运行中的应用交互，以及检查 Docker 对象（包括 Docker Compose 应用）。

## 容器操作

使用**搜索**字段可按名称查找特定容器。

在 **Containers** 视图中，您可以：
- 启动、停止、暂停、恢复或重启容器
- 查看镜像包和 **CVE**（公共漏洞与暴露）
- 删除容器
- 在 VS Code 中打开应用程序
- 在浏览器中打开容器暴露的端口
- 复制 `docker run` 命令以供重用或修改
- 使用 Docker Debug

## 资源使用情况

在 **Containers** 视图中，您可以监控容器随时间变化的 CPU 和内存使用情况。这有助于判断容器是否出现问题，或者是否需要分配更多资源。

当您检查容器时，**Stats** 选项卡会显示有关容器资源利用率的更多信息。您可以查看容器随时间使用了多少 CPU、内存、网络和磁盘空间。

## 检查容器

选择容器后，可以获取该容器的详细信息。

在这里，您可以使用快速操作按钮执行各种操作，例如暂停、恢复、启动或停止；也可以浏览 **Logs**、**Inspect**、**Bind mounts**、**Debug**、**Files** 和 **Stats** 选项卡。

### Logs（日志）

选择 **Logs** 可实时查看容器的输出。查看日志时，您可以：

- 使用 `Cmd + f` / `Ctrl + f` 打开搜索栏，查找特定条目。
  搜索匹配项会以黄色高亮显示。
- 按 `Enter` 或 `Shift + Enter` 分别跳转到下一个或上一个匹配项。
- 使用右上角的**复制**图标将所有日志复制到剪贴板。
- 显示时间戳
- 使用右上角的**清除终端**图标清空日志终端。
- 选择并查看日志中可能包含的外部链接。

您可以通过以下方式优化视图：

- 如果运行的是多容器应用程序，可以按特定容器筛选日志。
- 使用正则表达式或精确匹配搜索词。

### Inspect（检查）

选择 **Inspect** 可查看容器的底层信息。它会显示本地路径、镜像版本号、SHA‑256、端口映射及其他详细信息。

### Exec/Debug（执行/调试）

如果您在设置中未启用 **Docker Debug**，则会显示 **Exec** 选项卡。它允许您在运行中的容器内快速执行命令。

使用 **Exec** 选项卡相当于运行以下命令之一：

- `docker exec -it <container-id> /bin/sh`
- 当访问 Windows 容器时：`docker exec -it <container-id> cmd.exe`

如果您在设置中启用了 **Docker Debug**，或者在选项卡选项右侧切换打开了 **Debug mode**，则会显示 **Debug** 选项卡。

**Debug mode** 具有多个优势，例如：

- 可定制的工具箱。该工具箱预装了许多标准 Linux 工具，如 `vim`、`nano`、`htop` 和 `curl`。更多详情，请参阅 `docker debug` CLI 参考文档。
- 能够访问没有 shell 的容器，例如 slim 或无发行版容器（distroless containers）。

要使用 **Debug mode**：

- 将鼠标悬停在运行中的容器上，在 **Actions** 列下，点击 **Show container actions** 菜单。从下拉菜单中选择 **Use Docker Debug**。
- 或者，选择容器，然后选择 **Debug** 选项卡。

要默认使用 **Debug mode**，请导航至 **Settings** 中的 **General** 选项卡，然后选择 **Enable Docker Debug by default** 选项。

### Files（文件）

选择 **Files** 可浏览运行中或已停止容器的文件系统。您还可以：

- 查看哪些文件最近被添加、修改或删除
- 直接在内置编辑器中编辑文件
- 在主机和容器之间拖放文件和文件夹
- 右键单击文件即可删除不必要的文件
- 将容器中的文件和文件夹直接下载到主机
