# 容器化应用

本指南将带您逐步了解如何使用 Docker 容器化、运行、更新、共享和编排应用程序。您将掌握从单容器到多容器应用，再到使用 Docker Compose 进行服务编排的完整流程。

## 0. 准备应用脚本

在开始之前，您需要一个应用程序。我们以一个简单的 Node.js + Express 待办事项后端为例，它连接 MySQL 数据库并提供一个 REST API。

**目录结构示例：**

```
getting-started-app/
├── package.json
├── package-lock.json
├── src/
│   └── index.js
└── Dockerfile
```

**`package.json` 示例：**

```json
{
  "name": "docker-todo-app",
  "version": "1.0.0",
  "description": "A simple todo app",
  "main": "src/index.js",
  "scripts": {
    "start": "node src/index.js",
    "dev": "nodemon src/index.js"
  },
  "dependencies": {
    "express": "^4.18.2",
    "mysql2": "^3.6.0"
  },
  "devDependencies": {
    "nodemon": "^3.0.1"
  }
}
```

**`src/index.js` 示例：**

```javascript
const express = require('express');
const mysql = require('mysql2');

const app = express();
app.use(express.json());

const db = mysql.createConnection({
  host: process.env.MYSQL_HOST || 'localhost',
  user: process.env.MYSQL_USER || 'root',
  password: process.env.MYSQL_PASSWORD || 'secret',
  database: process.env.MYSQL_DB || 'todos'
});

app.get('/todos', (req, res) => {
  db.query('SELECT * FROM todos', (err, results) => {
    if (err) return res.status(500).send(err);
    res.json(results);
  });
});

app.post('/todos', (req, res) => {
  const { task } = req.body;
  db.query('INSERT INTO todos (task) VALUES (?)', [task], (err, result) => {
    if (err) return res.status(500).send(err);
    res.status(201).json({ id: result.insertId, task });
  });
});

const PORT = process.env.PORT || 3000;
app.listen(PORT, () => console.log(`App running on port ${PORT}`));
```

## 1. 构建镜像（Image）

使用 `Dockerfile` 定义如何构建应用镜像。在项目根目录创建 `Dockerfile`。

**示例 Dockerfile：**

```dockerfile
FROM node:18-alpine
WORKDIR /app
COPY package*.json ./
RUN npm ci --only=production
COPY . .
EXPOSE 3000
CMD ["npm", "start"]
```

- `FROM`：基础镜像（轻量级 Alpine Linux + Node.js）
- `WORKDIR`：设置工作目录
- `COPY`：复制文件
- `RUN`：执行命令（安装依赖）
- `EXPOSE`：声明容器监听端口（文档作用）
- `CMD`：容器启动时的默认命令

**构建镜像：**

```bash
docker build -t getting-started .
```

- `-t`：为镜像打标签（名称:标签，默认 latest）
- `.`：构建上下文路径（Docker 会查找该目录下的 Dockerfile）

> **提示**：使用 `.dockerignore` 文件排除 `node_modules`、`.git` 等，减少构建上下文大小。

## 2. 运行容器

```bash
docker run -d -p 127.0.0.1:3000:3000 --name todo-app getting-started
```

- `-d`：后台运行（detach 模式）
- `-p`：端口映射 `[主机地址:]主机端口:容器端口`。不指定主机地址时默认监听所有接口 `0.0.0.0`。
- `--name`：为容器指定名称，便于管理。

如果没有端口映射，主机无法访问容器内的应用程序。

**常用管理命令：**

```bash
docker ps                  # 列出运行中的容器
docker logs <container>    # 查看日志
docker stop <container>    # 停止容器
docker rm <container>      # 删除容器
```

## 3. 更新应用

镜像（Image）是不可变的，不能直接修改。当代码或配置变化时，需要：

1. 停止并删除旧容器：
   ```bash
   docker stop todo-app
   docker rm todo-app
   ```
2. （可选）删除旧镜像（如果不再需要）：
   ```bash
   docker rmi getting-started
   ```
3. 重新构建镜像：
   ```bash
   docker build -t getting-started .
   ```
4. 运行新容器。

> **最佳实践**：使用标签区分不同版本（如 `getting-started:v2`），便于回滚和版本管理。

## 4. 共享应用

将镜像推送到 Docker Hub 或其他镜像仓库，实现共享。

**步骤：**

1. 登录 Docker Hub：
   ```bash
   docker login
   ```
2. 为本地镜像打上仓库标签（格式：`<用户名>/<仓库名>:<标签>`）：
   ```bash
   docker tag getting-started yourusername/todo-app:latest
   ```
3. 推送镜像：
   ```bash
   docker push yourusername/todo-app:latest
   ```

其他人可通过 `docker pull yourusername/todo-app` 拉取并运行。

## 5. 持久化内容

容器停止或删除后，容器内部的文件变更会丢失。为了持久化数据（如数据库文件、配置文件、日志等），需要使用存储卷（Volumes）或绑定挂载（Bind Mounts）。

### 5.1 卷（Volume）

由 Docker 管理的存储单元，独立于容器生命周期，适合生产环境。

**创建卷：**

```bash
docker volume create todo-db
```

**运行容器时挂载卷：**

```bash
docker run -dp 127.0.0.1:3000:3000 \
  --mount type=volume,src=todo-db,target=/etc/todos \
  getting-started
```

- `src`：卷名称
- `target`：容器内挂载路径

**管理卷：**

```bash
docker volume ls          # 列出所有卷
docker volume inspect <卷名>  # 查看详情
docker volume rm <卷名>   # 删除卷（确保无容器使用）
```

### 5.2 绑定挂载（Bind Mount）

将主机上的任意目录或文件挂载到容器中，适用于开发环境的热加载、配置文件注入等。

**示例（开发模式，启用 nodemon 热重载）：**

```bash
docker run -dp 127.0.0.1:3000:3000 \
  --mount type=bind,src="$(pwd)",target=/app \
  -w /app \
  node:18-alpine sh -c "npm install && npm run dev"
```

- `src`：主机目录（建议使用绝对路径）
- `target`：容器内路径
- `-w`：覆盖工作目录

> **注意**：绑定挂载会直接使用主机文件，容器内修改也会影响主机。适合开发，但生产环境慎用（除非只读）。

### 5.3 两种方式对比

| 特性         | 卷（Volume）                | 绑定挂载（Bind Mount）           |
| ------------ | --------------------------- | -------------------------------- |
| 管理方式     | Docker 管理，位于 Docker 存储目录 | 用户管理，任意主机路径     |
| 适用场景     | 生产数据持久化、容器间共享   | 开发热重载、配置文件注入          |
| 跨平台兼容性 | 高（抽象层）                | 依赖主机文件系统路径              |
| 权限控制     | 自动处理                    | 需确保主机目录权限正确            |

## 6. 多容器应用

实际应用中，往往需要多个容器协同工作（如应用 + 数据库 + 缓存）。每个容器应该只做一件事。

### 6.1 容器网络

默认情况下，每个容器有独立 IP，可以通过 Docker 网络（Network）实现相互通信。

**创建网络：**

```bash
docker network create todo-app
```

**启动 MySQL 容器并连接到网络：**

```bash
docker run -d \
  --network todo-app --network-alias mysql \
  -v todo-mysql-data:/var/lib/mysql \
  -e MYSQL_ROOT_PASSWORD=secret \
  -e MYSQL_DATABASE=todos \
  --name mysql \
  mysql:8.0
```

- `--network-alias`：设置网络别名，其他容器可通过别名访问（类似 DNS）。
- `-e`：设置环境变量（数据库密码、数据库名等）。

**启动应用容器（连接到同一网络）：**

```bash
docker run -dp 127.0.0.1:3000:3000 \
  --network todo-app \
  -e MYSQL_HOST=mysql \
  -e MYSQL_PASSWORD=secret \
  -e MYSQL_DB=todos \
  getting-started
```

此时应用容器可通过主机名 `mysql` 访问数据库容器。

### 6.2 调试网络

使用 `nicolaka/netshoot` 工具容器排查网络问题：

```bash
docker run -it --network todo-app nicolaka/netshoot
```

进入容器后可使用 `ping mysql`、`nslookup mysql`、`telnet mysql 3306` 等命令。

## 7. 使用 Docker Compose

Docker Compose 允许您用一个 YAML 文件定义多容器应用，并通过一条命令启动/停止所有服务。

### 7.1 创建 `compose.yaml`

在项目根目录创建 `compose.yaml`：

```yaml
services:
  app:
    build: .
    ports:
      - "127.0.0.1:3000:3000"
    environment:
      MYSQL_HOST: mysql
      MYSQL_USER: root
      MYSQL_PASSWORD: secret
      MYSQL_DB: todos
    volumes:
      - ./:/app
    working_dir: /app
    command: sh -c "npm install && npm run dev"
    depends_on:
      - mysql

  mysql:
    image: mysql:8.0
    volumes:
      - todo-mysql-data:/var/lib/mysql
    environment:
      MYSQL_ROOT_PASSWORD: secret
      MYSQL_DATABASE: todos
    ports:
      - "3306:3306"   # 可选，如需主机访问

volumes:
  todo-mysql-data:
```

**说明：**

- `services`：定义各个服务（容器）。
- `app` 服务：
  - `build: .`：使用当前目录的 Dockerfile 构建镜像（无需单独 `docker build`）。
  - `ports`：端口映射。
  - `environment`：环境变量。
  - `volumes`：挂载卷（开发时绑定挂载当前目录实现热重载）。
  - `working_dir`：容器内工作目录。
  - `command`：覆盖默认 CMD。
  - `depends_on`：声明启动顺序（但不等待服务就绪，需应用内部实现重试）。
- `mysql` 服务：
  - `image`：直接使用官方镜像。
  - `volumes`：命名卷持久化数据库数据。
- `volumes` 顶层键：声明命名卷。

### 7.2 启动应用栈

```bash
docker compose up -d --build
```

- `-d`：后台运行。
- `--build`：启动前重新构建镜像。

**查看日志：**

```bash
docker compose logs -f         # 实时跟踪所有服务日志
docker compose logs -f app     # 仅跟踪 app 服务
```

**停止并删除所有容器、网络（但保留卷）：**

```bash
docker compose down
```

**同时删除卷：**

```bash
docker compose down --volumes
```

### 7.3 常用 Compose 命令

| 命令 | 说明 |
|------|------|
| `docker compose up` | 启动服务 |
| `docker compose down` | 停止并删除容器、网络 |
| `docker compose ps` | 列出服务状态 |
| `docker compose exec <service> <cmd>` | 在运行中的服务容器内执行命令 |
| `docker compose restart` | 重启所有服务 |
| `docker compose build` | 重新构建服务镜像 |

## 8. 构建镜像的最佳实践

良好的 Dockerfile 能显著减小镜像体积、提高构建速度并增强安全性。

### 8.1 利用构建缓存

**顺序优化**：将不易变化的指令放在前面，频繁变化的指令（如 COPY 源代码）放在后面。

```dockerfile
FROM node:18-alpine
WORKDIR /app
COPY package*.json ./
RUN npm ci --only=production   # 依赖变更时才重新安装
COPY . .                        # 源代码变更时只影响这一层
CMD ["npm", "start"]
```

### 8.2 多阶段构建

用于分离构建环境和运行环境，有效减小最终镜像尺寸。

**示例：构建一个 Go 静态二进制文件**

```dockerfile
# 阶段1：构建
FROM golang:1.21 AS builder
WORKDIR /src
COPY . .
RUN go build -o myapp .

# 阶段2：运行
FROM alpine:latest
COPY --from=builder /src/myapp /usr/local/bin/myapp
CMD ["myapp"]
```

最终镜像只包含二进制文件和 Alpine 基础系统，体积小巧。

**查看镜像（image）的层**：使用 `docker image history <镜像名>`

### 8.3 其他技巧

- 使用 `.dockerignore` 排除无用文件。
- 尽量使用官方 Alpine 版本基础镜像。
- 避免以 root 用户运行进程（创建应用专用用户）。
- 使用 `--no-cache` 或特定版本固定依赖，保证可重现构建。
