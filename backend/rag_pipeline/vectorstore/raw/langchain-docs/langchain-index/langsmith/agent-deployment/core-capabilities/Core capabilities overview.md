# 核心能力概述

> Agent Server 核心能力概述，包括流式传输、人机交互、MCP、A2A、分布式追踪、webhooks 和双文本。

Agent Server 提供了一套用于构建和运维生产级 agent 的能力。本节涵盖以下内容：

使用 LangGraph SDK 从已部署的 agent 实时流式传输输出。

暂停 agent 执行，以便在继续之前审查、编辑或批准工具调用。

从任意先前状态重放 agent 运行，以调试或探索替代路径。

将您的 agent 作为 MCP tools 暴露，供任何兼容 MCP 的客户端访问。

使用 A2A 协议实现 agent 之间的通信。

当从外部应用程序调用 Agent Server 时，统一跨服务的追踪。

响应来自已部署 agent 的运行事件，触发外部系统。

控制 Agent Server 在已有运行进行中时如何处理新消息。

### 持久化执行

其核心是，LangSmith Deployment 是一个持久化执行引擎。您的 agent 在托管任务队列上运行，并具有自动 checkpointing 功能，因此任何运行都可以从中断的确切点（而不是从头开始）进行重试、重放或恢复。

由于执行是持久化的，agent 可以执行在无状态运行时中可能脆弱或不可能完成的事情：

* **等待外部输入。** agent 调用 `interrupt()`，运行时 checkpoint 其状态，释放资源，并等待人类批准交易、审阅者编辑草稿或其他系统返回结果。当 `Command(resume=...)` 在数小时或数天后到达时，执行会从停止的地方精确恢复。这是人机交互工作流和时间旅行调试的基础。
* **在后台运行。** 后台运行无需阻塞调用者即可执行。运行时管理整个生命周期（排队、执行、checkpointing、完成），而客户端可以继续执行其他操作。
* **按计划运行。** Cron 作业以定期的节奏触发 agent 执行。每日摘要 agent、每周报告、定期数据同步。运行时按计划启动新的执行，并具有相同的持久化保证。
* **处理并发输入。** 当用户在 agent 运行中途发送新输入（双文本）时，运行时可以将其排队、取消正在进行的运行，或并行处理两者，而不会出现数据竞争或状态损坏。
* **失败时重试。** 可配置的重试策略控制退避、最大尝试次数以及在每个节点基础上触发重试的异常类型。运行能够在进程重启、基础设施故障以及执行中途的代码修订中存活下来。

有关容器、进程和任务队列如何协同工作的详细信息，请参阅 Agent Server：运行时架构。有关扩缩容和吞吐量调优，请参阅为扩缩容配置 Agent Server。
