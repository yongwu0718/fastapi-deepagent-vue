# 公众号爆文写作

## 概述

基于Vicky（文奇不惑）原创的《公众号变现25讲》18万字课程方法论 + 100+篇实战发文经验，打造的公众号爆文写作全流程技能。覆盖从选题到成稿的每一个环节，帮助创作者写出高原创度、低AI浓度、有个人温度的爆款文章。

**核心能力**：不是给你一堆模板让你套，而是给你一套方法论让你能持续产出高质量原创文章。

## 与爆文拆解skill的关系

本skill与`baowen-chaijie`（爆文拆解）skill配合使用：
- **爆文拆解** = 拆解一篇范文的爆款逻辑（输入一篇爆文 → 输出拆解报告）
- **爆文写作** = 基于拆解结果或独立选题，写出一篇原创爆文（输入选题/拆解报告 → 输出成稿）

两skill可独立使用，也可串联：先拆解一篇范文 → 再基于拆解结果仿写。

## 你的工作方式

顺序流程，7步成稿：

```
Step 1 确认写作意图 → 用户要写什么？什么类型？有没有参考范文？
    ↓
Step 2 选题策划 → 读取 references/xuanti.md → 确定选题方向和角度
    ↓
Step 3 标题设计 → 读取 references/biaoti.md → 设计3个候选标题
    ↓
Step 4 框架搭建 → 读取 references/kuangjia.md → 确定文章结构和细脉络
    ↓
Step 5 开头结尾设计 → 读取 references/kaijie.md → 设计开头和结尾
    ↓
Step 6 正文成稿 → 读取 references/chenggao.md → 完成正文写作
    ↓
Step 7 全文质检 → 原创度、AI浓度、铁律检查 → 输出最终成稿
```

每一步都有确认门——写完给用户看，用户说OK就下一步，说要改就修改。

## Step 1：确认写作意图

### 输入类型路由

| 用户输入 | 路由到 |
|----------|--------|
| "帮我写一篇公众号文章"+ 选题方向 | 直接进入Step 2选题策划 |
| 发了一篇爆文 + "仿写" | 先调用爆文拆解skill拆解，再进入Step 2 |
| 已有拆解报告 + "基于这个写" | 直接进入Step 3标题设计 |
| "写一篇干货文/观点文/清单体" | 进入Step 2，同时记录文章类型 |
| 只有模糊想法 | 进入Step 2，帮助用户从模糊想法中提炼选题 |

### 确认文章类型

从以下5种类型中确认（可让用户选择）：

| 类型 | 特征 | 适用场景 |
|------|------|----------|
| 认知观点文 | 输出一个颠覆认知的观点，用案例论证 | 个人成长、副业赚钱、职场认知 |
| 实操干货文 | 给出一套可执行的方法论/行动清单 | 方法论教学、技能分享、赚钱路径 |
| 清单体/金句体 | 以清单或金句为主体，简短有力 | 访谈提炼、读书金句、人生感悟 |
| 读书观点文 | 借一本书的观点，写出自己的人生解法 | 读书笔记、经典解读、书籍推荐 |
| 个人IP成长文 | 分享个人真实经历和成长感悟 | 个人故事、IP打造、成长记录 |

### 确认门

向用户确认：
> 写作意图已明确。确认以下信息：
> - 文章类型：[类型]
> - 选题方向：[方向]
> - 是否有参考范文：[有/无]
> 确认后进入选题策划。

## Step 2：选题策划

**必须先读取 `references/xuanti.md`** 获取完整的选题方法论。

### 核心流程

1. 帮用户从模糊想法中提炼选题方向和角度
2. 判断选题的覆盖人群、痛点和爆款潜力
3. 给出3个候选选题角度让用户选择

### 选题5维度评估

每个候选选题都要从5个维度评估：

| 维度 | 评估标准 |
|------|----------|
| 覆盖人群 | 这个选题覆盖多少读者？是否属于"大众刚需"？ |
| 痛点强度 | 截中了什么痛点？痛点是否足够普遍和强烈？ |
| 可操作性 | 读完能行动吗？有具体的"怎么做"吗？ |
| 情绪共鸣 | 读完会有什么情绪反应？共鸣/爽感/焦虑？ |
| 可迁移性 | 这个选题框架能换领域复用吗？ |

### 确认门

向用户展示3个候选选题（含5维度评估），让用户选择后进入标题设计。

## Step 3：标题设计

**必须先读取 `references/biaoti.md`** 获取完整的标题方法论和句式库。

### 核心流程

1. 基于选定选题，设计3个候选标题
2. 每个标题标注使用的技巧类型和关键词
3. 让用户选择或组合

### 标题设计5要素

| 要素 | 说明 |
|------|------|
| 技巧类型 | 反常识/悬念/展示回报/身份锁定/超级符号借势（至少选2种叠加） |
| 吸睛关键词 | 数字/痛点词/情绪词/超级符号IP/身份词 |
| 表达句式 | 从句式库中选取或创新 |
| 覆盖人群 | 标题是否让目标人群一眼对号入座 |
| 点击冲动 | 3秒内是否能让人忍不住点进来看 |

### 确认门

向用户展示3个候选标题（含技巧分析和迁移句式），让用户选择后进入框架搭建。

## Step 4：框架搭建

**必须先读取 `references/kuangjia.md`** 获取8种通用结构范式和搭建方法。

### 核心流程

1. 根据文章类型选择最合适的结构
2. 还原搭建思路：角度→拆分维度→小标题→段落关系
3. 设计细脉络：每个模块写什么→先写什么→后写什么

### 结构选择指引

| 文章类型 | 推荐结构 | 说明 |
|----------|----------|------|
| 认知观点文 | 观点三步法（总分总）或 逻辑递进 | 总论点→分论据→总结升华，或层层深入 |
| 实操干货文 | 干货三件套2W1H | 是什么→为什么→怎么做 |
| 清单体/金句体 | 同维度并列拆分 或 清单体 | 多个平行角度展开 |
| 读书观点文 | 清单体+个人感悟 | 书中核心点+个人应用 |
| 个人IP成长文 | 时间线 或 冲突-解决 | 起点→转折→结果→启示 |

### 确认门

向用户展示完整框架（大框架+细脉络+小标题），确认后进入开头结尾设计。

## Step 5：开头结尾设计

**必须先读取 `references/kaijie.md`** 获取开头结尾技巧、金句模板和句式库。

### 开头设计

1. 选择开头模式（从5种高频模式中选取）
2. 按行文路径设计：先写什么→再写什么→最后写什么
3. 确保开头在3句内抓住注意力

### 结尾设计

1. 选择结尾模式（从4种高频模式中选取）
2. 按行文路径设计：总结→升华/呼吁/行动
3. 确保结尾有力，激发转发或行动

### 确认门

向用户展示开头和结尾设计，确认后进入正文成稿。

## Step 6：正文成稿

**必须先读取 `references/chenggao.md`** 获取素材方法论、写作流程和成稿规范。

### 核心流程

1. 搜集素材：按素材方法论确定素材来源和使用方式
2. 逐段写作：按细脉络框架，逐模块完成正文
3. 穿插表达手法：在每个模块中有意识地使用10种表达手法
4. 穿插金句：在关键位置放置金句（开头后/转折处/结尾前）

### 写作铁律（Vicky铁律，不可违反）

1. **原创度必须超85%** — 不能照搬范文的句子和段落，必须用自己的话重新表达
2. **AI浓度低于5%** — 禁止复杂句型，禁止"不是……而是……"、"真正的……是/不是……"、"你是不是也……"等句型
3. **全文不允许用加粗强调某个词** — 这些会被识别为AI创作
4. **开头必须场景代入痛点** — 废除模板开头，坚决不要"你是不是也…"和"很多人都…"这种生拉硬拽的方式；最好的开头像讲故事，用真人真事的场景代入
5. **语言口语化有温度** — 像朋友聊天一样的语气，不要书面腔
6. **短句优先** — 能用短句说清的不用长句，短句有力量感
7. **每个段落有且只有一个核心信息** — 不要一段话塞多个观点

### 素材使用原则

- 每个观点至少配1个素材（案例/数据/引用）
- 素材来源要多元：书本+个人经历+第三方佐证，不要通篇只用同源素材
- 案例必须缩写：提炼关键元素（谁+做了什么+结果），不要大段搬运原文
- 案例和观点之间要有论证逻辑衔接

### 确认门

写完全文后，向用户展示完整成稿，进入质检环节。

## Step 7：全文质检

### 质检清单

| 检查项 | 标准 | 通过？ |
|--------|------|--------|
| 原创度 | >85%，无照搬范文句子 | |
| AI浓度 | <5%，无禁止句型 | |
| 加粗使用 | 全文无加粗强调某个词 | |
| 开头质量 | 3句内抓住注意力，场景代入 | |
| 标题吸引力 | 3秒内有点击冲动 | |
| 结构清晰 | 大框架+细脉络可追溯 | |
| 素材搭配 | 观点有案例支撑，来源多元 | |
| 金句分布 | 至少3-5条金句分布在关键位置 | |
| 结尾有力 | 有行动呼吁/升华/共鸣收束 | |
| 语言风格 | 口语化、有温度、短句为主 | |

### 如果质检不通过

- 原创度不够 → 重写相关段落，用自己的话重新表达
- AI浓度高 → 替换禁止句型，简化复杂句式
- 开头太模板 → 换一个场景代入的开头

### 最终交付

质检通过后，输出最终成稿。告知用户：
> 文章已写完，质检通过。你可以：
> 1. 直接复制到公众号编辑器
> 2. 说"调整XX部分"进行修改
> 3. 说"换一个标题"重新设计标题
