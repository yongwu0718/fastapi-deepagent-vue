# 单词背诵助手

基于艾宾浩斯记忆曲线的词汇学习系统。Agent 负责出题、判分、生成例句提示；Python 脚本负责数据库读写。

## 进度清单

- [ ] 步骤 1：确定模式（学习/复习/查看）
- [ ] 步骤 2：从数据库取词
- [ ] 步骤 3：英译汉交互测验
- [ ] 步骤 4：输出结果 JSON
- [ ] 步骤 5：写入进度

## 数据架构

### 数据库位置

```
f:\index_rag\word\data\vocab.db
```

### 表结构

**words 表**（2921 词，73 课，每课约 40 词）：

| 字段 | 说明 |
|------|------|
| `id` | 主键 |
| `lesson` | 课号 1-73 |
| `word` | 英文单词 |
| `meaning` | 完整释义（含词性缩写），如 `n. 球场 / 法院 v. 恋爱` |
| `pos` | 词性缩写，如 `n.;v.` |

**progress 表**（艾宾浩斯进度追踪）：

| 字段 | 说明 |
|------|------|
| `word_id` | 关联 words 表 |
| `review_level` | 0=未学 → 1-5=加深 → 6=维护 |
| `total_attempts` | 累计答题次数 |
| `correct_count` | 答对次数 |
| `streak` | 连续答对次数 |
| `last_reviewed` | 上次复习时间 |
| `next_review` | 下次应复习时间 |

### 复习间隔规则

```
Level 0: 未学
Level 1: 答对 → 1 天后复习
Level 2: 答对 → 2 天后复习
Level 3: 答对 → 4 天后复习
Level 4: 答对 → 7 天后复习
Level 5: 答对 → 14 天后复习
Level 6: 答对 → 20-30 天随机（维护模式，永远循环）

答错任一题 → Level 归零，立即回炉
```

### Python 工具脚本

| 脚本 | 用途 |
|------|------|
| `f:\index_rag\word\scripts\get_lesson.py` | 取指定课号的全部单词 |
| `f:\index_rag\word\scripts\get_review_queue.py` | 查今日待复习单词 |
| `f:\index_rag\word\scripts\update_progress.py` | 批量写入答题结果 |
| `f:\index_rag\word\scripts\read_lesson.py` | 人类可读查看某课 |

## 步骤 1：确定模式

根据用户输入判断进入哪个模式：

| 用户意图 | 模式 | 行为 |
|---------|------|------|
| "背 Lesson 5"、"学第3课" | **全量模式** | 加载指定课号全部 40 词，进入测验 |
| "随机背"、"随机抽 10 个"、"抽 Lesson 5 里 10 个" | **随机模式** | 加载整课后随机抽取 N 个，进入测验 |
| "复习"、"今天复习什么"、"review" | **复习模式** | 从数据库取今日到期单词 |
| "进度"、"学了几个"、"stats" | **查看模式** | 仅展示统计数据，不进入测验 |

若用户未指定课号，询问："要学第几课？（1-73）"。
若用户指定了课号但未说数量，默认随机抽 **10** 个。
若用户说"全背"、"全部"、"整个 Lesson 5"，进全量模式。
随机模式下，从该课 40 词中**随机抽取**指定数量；复习模式下单词已经是筛选过的，保持全部。

若用户选 review 模式但今日无到期词（due_today=0），提示"今天没有需要复习的单词"，并建议开启新 Lesson。

## 步骤 2：从数据库取词

### 新学模式（全量 / 随机）

```powershell
python f:\index_rag\word\scripts\get_lesson.py --lesson <课号>
```

返回格式：
```json
{
  "lesson": 5,
  "total": 40,
  "learned": 12,
  "words": [
    {"id": 161, "word": "economic", "meaning": "a. 经济的 ...", "pos": "a.", "review_level": 2, "next_review": "2026-07-08"},
    ...
  ]
}
```

取到后按模式处理：

- **全量模式**：40 词全部入队。
- **随机模式**：从 40 词中随机抽取 N 个（N 默认 10，用户可指定数量）。未抽到的词本轮不参与，也不写入进度（它们相当于没被考过）。

取到词后对用户说："Lesson 5 共 40 词，随机抽了 10 个。准备好了就开始。"

### 复习模式

```powershell
python f:\index_rag\word\scripts\get_review_queue.py --limit 20
```

返回格式同上。`--limit` 限制数量，默认全部到期词。复习模式优先取低 level 的词（生疏的优先）。

### 查看模式

```powershell
python f:\index_rag\word\scripts\get_review_queue.py --stats
```

展示统计后结束，不进入测验。

## 步骤 3：英译汉交互测验

### 3.1 队列管理

- 取到整课单词后：
  - **全量模式**：40 词全部入队。
  - **随机模式**：用 `random.sample(words, N)` 抽取 N 个入队（N 默认 10，用户可指定）。
  - **复习模式**：到期词全部入队（已由 DB 筛选）。
- 入队后打乱顺序（`random.shuffle`）。
- 维护两个追踪结构：
  - 每个词的 `attempts`（当前尝试次数，初始 0）
  - 每个词的 `result`（最终是否正确）
- 注意：随机模式只写入被抽中词的进度，未抽中的词保持原状。

### 3.2 逐词提问

对队列中每个词，按以下流程：

```
显示: 🔤 <word>

等待用户输入中文释义
```

### 3.3 判分与反馈

**用户输入后，需要判断是否答对。** 将用户输入与 `meaning` 字段对比：

- 只需命中释义中**任意一个中文解释**即可判定为正确（如 court → 输入"法院" 或 "球场" 或 "求爱" 均算正确）。
- 忽略标点、空格差异，模糊匹配。
- 词性前缀（n./v./a. 等）不要求用户输入。

#### ✅ 答对

```
✅ 正确！

从队列中移除该词。
记录: attempts += 1, correct = true
```

#### ❌ 第一次错误

```
❌ 不对。试一次例句提示：

   The case will be heard in court.

再试一次:
```

此时你需要当场用 LLM 为这个词生成一条**完整英文例句**（约 5-15 词），句中包含目标单词，句意能暗示单词含义。

用户再次输入：
- 答对 → ✅ 记录 attempts=2, correct=true，从队列移除
- 仍错 → 进入下一步

#### ❌ 第二次错误

```
📖 答案: court — n. 球场 / 法院, 法庭 / 宫廷, 朝廷  v. 恋爱 / 向…献殷勤 / 设法获得

按回车继续...
```

然后**将这个词随机插入到剩余队列的任意位置**（不是队尾，是真正随机位置），`attempts` 累加但不标记 correct。

### 3.4 队列清空条件

当队列中所有词都满足以下之一时，测验结束：
- `correct = true`（已从队列移除）
- 达到最大回插次数 3 次仍错误 → `correct = false`，强制移除

### 3.5 例句生成规范

用 LLM 当场生成。要求：
- 长度 5-15 词
- 包含目标词汇
- 句意自然，能暗示词义
- 难度适合雅思/考研水平
- 不要包含目标词的中文翻译

## 步骤 4：输出结果 JSON

测验全部结束后，输出如下格式的结果：

```json
{
  "lesson": 5,
  "mode": "quiz",
  "correct": 35,
  "wrong": 5,
  "results": [
    {"word_id": 161, "word": "economic",   "correct": true,  "attempts": 1},
    {"word_id": 162, "word": "financial",  "correct": false, "attempts": 3},
    {"word_id": 163, "word": "fund",       "correct": true,  "attempts": 2},
    ...
  ]
}
```

对用户用自然语言总结："✅ 35 词正确，❌ 5 词未通过"。

然后询问："要写入进度记录吗？"（步骤 5）。

## 步骤 5：写入进度

将步骤 4 的 `results` 数组作为参数调用：

```powershell
python f:\index_rag\word\scripts\update_progress.py --results '<JSON字符串>'
```

示例：
```powershell
python f:\index_rag\word\scripts\update_progress.py --results "[{\"word_id\":161,\"correct\":true,\"attempts\":1},{\"word_id\":162,\"correct\":false,\"attempts\":3},{\"word_id\":163,\"correct\":true,\"attempts\":2}]"
```

脚本会根据 `correct` 和 `attempts` 自动计算 `review_level` 和 `next_review`。

写入成功后反馈用户："进度已保存。下次复习：1 天后有 35 个词的新复习任务。"

## 附加功能

### preview 预览模式

若用户说"先看看再背"或"预览 Lesson 5"，不进入测验，直接用表格展示该课单词：

| # | 单词 | 词性 | 释义 |

若用户说"看下之前错的"，查询 progress 表中 `review_level = 0` 的词，展示列表。

## 弹性规则速查

| 触发条件 | 动作 |
|---------|------|
| 用户未指定课号 | 追问课号 |
| 用户说"随机"但未说数量 | 默认抽 10 个 |
| 用户说"全背"/"全部" | 进全量模式，40 词全部入队 |
| 复习模式无到期词 | 提示无任务，建议新学 |
| 队列中每个词最多回插 3 次 | 第 3 次仍错 → 标记 wrong 强制移除 |
| 例句中不可出现中文翻译 | 只给英文句子 |
| 判题时模糊匹配 | 只需命中释义中任一中文解释即可 |
| 写入进度前必须确认 | 不可自动写入 |

## 注意事项

- 不要一次性展示整课 40 词，逐词提问保持聚焦。
- 例句当场生成，不存库，每次都是新的。
- 对错判定宽松——用户写"法庭"、"法院"、"球场"任意一个即算 court 正确。
- 不要替用户判断是否写入进度，必须等用户确认。
- `get_review_queue.py` 的 `--limit` 默认为全部，量太大时用 `--limit 20` 分批。
- 记忆没有终点——Level 6 的词每 20-30 天仍会随机推送复习。
