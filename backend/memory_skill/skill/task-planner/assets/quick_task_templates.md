# 随手池模板

随手池用于收纳满足以下全部条件的微任务：**<15 分钟 + 单一动作 + 无依赖 + 当天可完成**。
不占周池容量，不参与重要性排序，每日清空。

## 输出格式

输出随手池时使用以下 Markdown 表格，与每日计划时间块一并展示：

```markdown
### 📥 随手池
<!-- 微任务：<15分钟 + 单一动作 + 无依赖，不占周池容量，不参与重要性排序。每日清空。 -->
| # | 任务 | 时长 | 状态 |
|---|------|------|------|
| 1 | 取快递 | 10min | 📋 |
| 2 | 回复XX消息 | 5min | 📋 |
| 3 | 填报销单 | 10min | 📋 |
```

## 准入规则

| 条件 | 要求 | 不满足时处理 |
|------|------|-------------|
| 耗时 | < 15 分钟 | 退回步骤 3 作为日行动拆解 |
| 动作 | 单一动词 + 单一对象 | 拆分后再判断 |
| 依赖 | 无前置依赖 | 标记依赖后走标准通道 |
| 窗口 | 当天可完成 | 不进入随手池，归入对应日期的日计划 |

## 状态图标

| 图标 | 状态 |
|------|------|
| 📋 | 待执行 |
| ✅ | 已完成 |
| ❌ | 今日未完成（日终自动丢弃，不迁移） |

## 运行规则

| 规则 | 说明 |
|------|------|
| **不入周池** | 随手池独立于周池，不参与重要性排序和容量计算 |
| **每日清空** | 日终未完成的随手任务不迁移到次日（若仍需要，次日手动重新添加） |
| **间隙填充** | 每日计划时间块之间若有 ≥15 分钟间隙，优先从随手池拉取填充 |
| **运行时入口** | 步骤 5.4 增量添加时，满足准入条件的直接丢入随手池 |

## 验证规则

| 检查项 | 通过标准 | 标记 |
|--------|---------|------|
| 准入合格 | 池中每条任务满足 4 条准入条件 | ✅/⚠ |
| 无超大任务 | 无 ≥15 分钟的任务混入 | ✅/⚠ |
| 日均清空 | 昨日随手池已清空，无残留任务 | ✅/⚠ |
| 总量合理 | 随手池总计 ≤ 60 分钟（避免替代真正的时间块规划）| ✅/⚠ |

> 随手池不是"低优先级任务垃圾桶"，而是"见缝插针即可完成的微小事项"。如果随手池长期超过 10 条，建议审视是否有任务实际需要列入日计划。
