# 周视图规划指南

当用户需要详细周视图时加载本指南。任务拆解和日计划生成已在 SKILL.md 主工作流中完成，本指南聚焦于周视图特有的构建逻辑和输出格式。

## 何时切换为周视图

- 用户明确要求查看"本周整体安排"或"周计划总览"。
- 本周任务涉及跨日依赖链，需要可视化每一天的负载。
- 用户想从周视角审视弹性任务的分配是否合理。

## 周视图构建

主工作流（SKILL 步骤 3 级联拆解 → 步骤 5 构建周池）已完成周任务的拆解和重要性排序。本指南在此之上补充周视图特有的构建步骤：

### 步骤 1：确定本周三大目标

引导用户从当前周池中提取本周最关键的 3 个目标。若用户不确定，提问："如果本周只能完成三件事来换取满意度，这三件是什么？"

### 步骤 2：每日主题分配

- 为每天设定一个主题（如"深度工作日"、"会议日"、"弹性日"），避免每天混合所有类型。
- 将三大目标对应的任务优先排入主题匹配的日子。
- 注意避开用户的固定占用时间（参考 `energy_mapping.md` 的个性化精力地图）。

### 步骤 3：标注跨日任务链

注意 SKILL 步骤 3 已完成的级联拆解结果中可能包含跨日任务链。在周视图中：
- 标注跨日任务链中每日子步骤的执行状态和依赖。
- 若某日未完成导致后续链断裂，触发弹性调整（回周池按重要性重排）。

### 步骤 4：区分核心与弹性

将周池中的任务分为两组展示：

| 分组 | 来源 | 处理 |
|------|------|------|
| 本周核心 | 硬截止 / 阻塞链前端 / SKILL 步骤 5.1 归入核心的 | 不推至下周 |
| 周池弹性 | 重要但无硬截止 / SKILL 步骤 5.1 归入弹性的 | 超额时删减，回到长期计划月度里程碑 |

### 步骤 5：校验负载均衡

- 确保每日总任务时长不超过 4.8h，且深度任务不连续超过 3 小时。
- 每日总时长之间相差不超过 1.5h，避免某天过载。
- 周池弹性任务占比不超过 40%，超出说明本周容量不足，需与用户协商。

### 输出格式

构建完成后，读取 `assets\weekly_templates.md`，按模板格式输出 Markdown 表格。逐项执行模板中的验证规则并标注通过状态（✅/⚠）。
