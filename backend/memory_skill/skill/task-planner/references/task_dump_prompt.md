# 任务收集引导提问

请按**时间跨度从大到小**的顺序，自顶向下引导用户。**必须完整走完所有阶段，整理出完整的时间层级链条后，才能进入步骤 2。严禁在收集阶段讨论任何日计划的细节分配（如休息几分钟、精力匹配、时段安排）。**

**核心原则：AI 先构建，用户再确认。** 不要抛出开放性问题让用户自己想——Agent 应根据已收集到的信息和领域常识主动构建提案，用户只需确认或纠正。只有关键信息确实缺失时才追问（用最小化追问，见各阶段末尾）。

---

## 阶段一：长期目标（>3 个月）

从最高视角开始，先锁定大方向。

**模式**：用一个开放性问题获取用户的目标方向，然后根据回答**主动推断**出领域、进度、截止日期等，以结构化卡片回显给用户确认。

**开场**：
"未来半年到一年，你最重要的目标是什么？不用想具体怎么做，先说是什么。"

**Agent 动作**：用户回答后，根据关键词（考研/考公/跳槽/学技能等）**由 Agent 主动推断**目标领域、可能的子项、常见进度状态，以表格回显。例如用户说"考研"，Agent 应主动列出可能的科目矩阵并标注已知进度。

**最小化追问**（仅在完全无法推断时提问）：
- 有没有硬截止日期？
- 有没有已经完成/正在推进/还没开始的区分需要你补充的？

---

## 阶段二：中期目标（1-3 个月）

把长期目标拆到可感知的季度/月度里程碑。

**模式**：Agent 根据阶段一的长期目标 + 截止日期，**主动构建**一个 1-3 个月的时间线表格，包含各子项的阶段性节点和交付物。用户只需说"对"或"调整某处"。

**Agent 动作**：
- 根据目标类型（考试/项目/学习等）和截止时间，按常识倒推合理的时间阶段划分（如「基础收尾→强化→真题→冲刺」）。
- 将用户已提到的进度填入各时间段。
- 以「总时间线 + 当前月度」两层展示。

**最小化追问**（仅关键参数缺失时追问，每次不超过 2 个问题）：
- Agent 提案中无法推断的具体工具/资料（如"高数强化用哪本题库？660还是880？"）
- 与常规节奏明显不符的节点（如专业课迟迟未启动是否需要调整）

---

## 阶段三：月度里程碑

把中期目标落实到当前月份。

**模式**：Agent 根据阶段二的时间线，**自动生成**当前月的各科月度目标表，包含预估产出和每周推进节奏。

**Agent 动作**：
- 从阶段二的时间线中截取当前月份。
- 按科目/子项展开为每周推进量（如"线代每天看X讲视频，预计N周完成"）。
- 标注本月不可错过的关键节点（如报名截止、真题启动日）。

**最小化追问**（仅无法合理推断时）：
- 本月有没有 Agent 不知道的外部节点？（如报名、确认、考试）

---

## 阶段四：周任务

从月度拆到本周（当前自然周）。

**模式**：Agent 根据阶段三的月度目标 + 精力地图（步骤 0 已收集），**主动生成**本周任务清单，分为硬截止和弹性两类，用户只需调整优先级或纠正任务量。

**Agent 动作**：
- 从月度目标中提取本周应推进的任务。
- 自动标注硬截止 vs 弹性。
- 根据各科视频/书籍典型时长和用户可用时间，给出每条任务的预估耗时。

**最小化追问**（仅在数据不足以拆分时）：
- 确认本周是否有阶段一中未提及的临时事件打乱节奏。

## 阶段五：领域补充（可选，快速扫描）

完成自顶向下收集后，仅快速扫描以下领域，确认无遗漏。此阶段**不展开讨论**，只做是否判断题。

- 个人事务（缴费、办证、购物）——有吗？
- 重复例行（每日复盘、锻炼）——有吗？
- 模糊事项（改天有空做但一直没安排）——有吗？
- 当前阻塞（等待他人/他人等待你）——有吗？

无遗漏则直接进入阶段六。

## 阶段六：输出结构化任务树

将收集到的所有任务，按时间层级整理为：

```
🟣 长期目标 → 🟠 中期（1-3月）→ 🟡 月度 → 🔵 周级 → ✅ 日行动
```

**输出格式**：

```markdown
## 收集完成 · 任务树

### 🟣 长期（>3月）
- 目标A：... | 截止：... | 进度：...%

### 🟠 中期（1-3月）
- 节点1：... | 截止：... | 进度：...%
  - 依赖：...
- 节点2：...

### 🟡 本月
- 周目标1：... | 产出：...
- 周目标2：...

### 🔵 本周
- [硬截止] 任务1 | 预估耗时：...
- [弹性] 任务2 | 预估耗时：...

### ✅ 日行动（暂不细化，步骤 5 填充分配）
```

完成后，将结构化任务树交给步骤 2（按时间跨度分类），开始进入下一步。

## 对 Agent 的引导说明

- **AI 先构建，用户后确认是硬规则**：永远不要让用户自己想答案。Agent 应基于已收集信息和领域常识主动构建提案，用户只需说"对"或"调整某处"。
- **自顶向下是硬规则**：必须先走完阶段一到四，才能进入阶段五。任何时候不得在收集阶段追问日计划细节。
- **禁止日计划纠缠**：收集阶段绝不允许讨论「休息几分钟」「哪个时段做什么」「精力匹配」等步骤 5 才处理的内容。发现自己在讨论这些 → 立即停止，返回阶段一中尚未完成的层级。
- **最小化追问**：追问只用于填补 Agent 无法推断的关键缺口。每次追问不超过 2 个问题。能用常识推断的（如考研科目、复习阶段划分、典型教材）直接推断。
- 若用户给出的是日级零散任务，引导反问：「这些任务服务于哪个更大的目标？」将其上溯到正确的层级。
- 若用户跳过某个时间层级，记录为「该层级暂不明确」并在下一层级继续，不要编造。
- 收集过程中不排序、不评判重要性、不讨论可行性。
