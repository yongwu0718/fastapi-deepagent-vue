# 可用时间窗口抽取

## 原则

一次性收集用户的可用时间窗口，不做深度追问、不聊心理状态、不讨论任务匹配。**只收集时间结构，精力标注为可选追问。** 三步完成，不超过两轮对话。收集内容仅供步骤 5 构建每日计划时使用，收集阶段不再展开讨论。

## 知识参考：四种精力类型

以下定义在步骤 5 构建日计划时使用。收集阶段不展开四种类型的匹配讨论。

| 类型 | 特征 | 适合安排 |
|------|------|---------|
| 🔵 深度专注 | 高认知负荷，需持续注意力 | 写作、编程、数学、复杂分析 |
| 🟡 沟通协作 | 需社交互动与实时反应 | 会议、讨论、面试、评审 |
| ⚪ 机械执行 | 低认知，重复性操作 | 填表、归档、清理、整理 |
| 🟢 学习充电 | 需理解吸收新信息，无即时产出压力 | 阅读、看视频、听课、研究 |

---

## 阶段一：时间窗口抽取（必须一次性完成）

引导用户提供三个维度的数据。用一句整合提问完成，不加追问轮：

**整合提问**："告诉我你一天的时间结构——
1. 几点能开始干活？几点必须结束？
2. 中间哪些时段被固定占用（吃饭、午觉等）？
3. 哪些时段是高效段、哪些是低效段？（只说自己确定的）"

**输出要求**：收集完毕后直接生成时间窗口表，不做任何讨论。

---

## 阶段二：输出时间窗口表

```markdown
| 时段 | 状态 | 可排任务 |
|------|------|---------|
| HH:MM - HH:MM | 🔋 高效 / 🔻 低效 / ⛔ 固定占用 / — 休息 | （步骤 5 填充） |
```

**固定占用时段**（⛔）：饭点、午觉等不可占用时间，不排任何任务。

**休息时段**（—）：22:00 后等，绝对不排任务。

"可排任务"列在收集阶段留空，由步骤 5 按精力类型匹配填充。

**若用户无法提供精力标注**（不确定高效/低效段），只填充状态为「⛔ 固定占用」和「— 休息」的时段，其余标记为「可用」，步骤 5 使用下方默认双峰模型填充。

---

## 阶段三：默认后备方案

若用户未标注精力高低，使用以下默认模型：

| 时段 | 状态 | 说明 |
|------|------|------|
| 08:00-10:00 | 🔋 高效 | |
| 10:00-12:00 | 🔋 中高 | |
| 12:00-13:00 | ⛔ 固定 | 午餐 |
| 13:00-15:00 | 🔻 低效 | |
| 15:00-17:00 | 🔋 中高 | |
| 17:00-19:00 | 🔻 低效 | |
| 19:00-22:00 | 🔋 中高 | |
| 22:00 以后 | — 休息 | |

用户的固定占用时段**始终覆盖**默认模型中的对应时段。

---

## 对 Agent 的引导说明

- **一次收集、一步到位**：用一句整合提问获取全部时间结构数据。禁止拆成多轮追问。
- **禁止精力追问**：不追问"你早上状态怎么样？午后犯困吗？晚上还有精力吗？" 用户说了就说，没说就用后备方案。
- **禁止讨论任务匹配**：不讨论"深度任务放在上午还是下午"——这是步骤 5 的事。
- 收集完毕后立即生成时间窗口表 → 交给步骤 5 使用。表生成后不允许对表本身做任何讨论。
- 若用户说"我不知道哪个时段高效"，直接使用后备方案，不加追问。
