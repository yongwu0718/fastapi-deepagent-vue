## 路由设计

| 路径 | 组件 | 说明 |
|------|------|------|
| `/` | （重定向） | 自动跳转到上次活跃线程或新建线程 |
| `/chat/:threadId` | `ChatLayout.vue` | 主聊天视图（懒加载） |
| `/settings` | `SettingsView.vue` | 设置管理页面（懒加载） |
| `/rag` | `RagManagement.vue` | RAG 向量库管理页面（懒加载） |

- 使用 `createWebHashHistory()` Hash 模式路由
- 根路径 `/` 通过 `localStorage.getItem('chat_active_thread_id')` 恢复上次活跃线程

## 核心数据流

```
用户输入 (ChatInput.vue)
  │
  ├── 发送消息 → useChatController.sendMessage()
  │     ├── 新建线程（若 threadId 为空）
  │     ├── 构建消息（user role + contentBlocks）
  │     ├── 添加到 state.messages
  │     └── 委托 → useChatStream.sendMessage()
  │           ├── 有附件 → doSseFormDataRequest (multipart)
  │           └── 无附件 → doSseRequest (JSON)
  │                 └── SSE 事件流
  │                       ├── chunk 事件 → sseChunkHandler
  │                       │   ├── reasoning → streamingReasoning
  │                       │   ├── text → streamingContent
  │                       │   ├── tool_call → pendingToolCalls
  │                       │   ├── tool_result → 更新 toolCalls
  │                       │   ├── checkpoint → 绑定检查点ID
  │                       │   ├── interrupt → showInterrupt
  │                       │   ├── rubric → Toast 通知
  │                       │   └── done → 打包为 assistant 消息
  │                       └── 工具调用同步 → useToolMessages → RightSidebar
  │
  └── 取消 → useChatStream.cancelRequest()
        └── abort + 打包未完成内容为 assistant 消息
```

## 设计模式与约定

| 模式 | 应用位置 | 说明 |
|------|----------|------|
| **Composition API 单例** | `useFileManager`, `useToolMessages` | 模块级唯一实例，跨组件共享状态 |
| **发布-订阅** | `useToast` | 解耦 Toast 发送方和渲染方 |
| **Controller 模式** | `useChatController` | 编排 state + stream + checkpoints 三个子模块 |
| **SSE 事件驱动** | `sseChunkHandler` | 10 种事件类型的状态机处理 |
| **localStorage 双重同步** | `useChatHistory` | 线程状态与 Vue Router 双向同步 |
| **懒加载路由** | `router/index.ts` | 所有页面组件均为动态 import |

## 主题系统

使用 CSS 变量实现亮色/暗色双主题，通过 `prefers-color-scheme` 媒体查询自动切换：

```css
:root {
  --text: #6b6375;          /* 正文颜色 */
  --text-h: #08060d;        /* 标题颜色 */
  --bg: #fff;               /* 背景色 */
  --border: #e5e4e7;        /* 边框颜色 */
  --code-bg: #f4f3ec;       /* 代码块背景 */
  --accent: #aa3bff;        /* 强调色 */
  --accent-bg: rgba(...);   /* 强调色背景 */
  --sans: system-ui, ...;   /* 无衬线字体 */
  --mono: ui-monospace, ...;/* 等宽字体 */
}
```

## 全局错误处理

在 `main.ts` 中通过 `app.config.errorHandler` 和 `warnHandler` 捕获全局 Vue 错误，统一通过 `loggerVue.error/warn` 输出。
