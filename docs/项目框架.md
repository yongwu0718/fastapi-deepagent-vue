### 后端架构

后端采用分层架构：**API 路由层 → 服务层 → Agent 核心层 → 工具/数据层**。

- **API 层** (`backend/api/`)：7 个路由模块，统一异常处理，Pydantic Schema 校验
- **Agent 核心层** (`backend/core/`)：Agent 编译、模型工厂、提示词、中间件、技能管理、子智能体、工具集
- **RAG 管道** (`backend/rag_pipeline/`)：文档清洗、PDF 加载、Markdown 分块、向量入库
- **配置层** (`backend/config/`)：环境变量、日志系统、Langfuse 可观测性
- **CLI** (`backend/cli/`)：命令行交互入口，含流处理、状态保存、聊天日志

### 前端架构

前端采用模块化组合式架构，全部组件自研（无第三方 UI 库）。

- **聊天核心** (`src/chat/`)：控制器 + 状态 + 流式通信 + SSE 处理器
- **文件管理** (`src/files/`)：浏览器、预览、增删改、Markdown 渲染
- **线程管理** (`src/threads/`)：对话线程创建/切换/删除/历史缓存
- **RAG 管理** (`src/rag/`)：向量库入库（预览→确认）、文档删除、配置管理、健康监控
- **侧边栏** (`src/sidebar/`)：工具调用 Tab、大纲导航、详情 Tab
- **设置** (`src/settings/`)：6 Tab 配置编辑器（Markdown/YAML/JS）

## 功能模块详解

### 1. 两阶段 RAG 知识库检索

Agent 通过 `retriever_row_doc_tool` 工具执行检索：

1. **向量召回**：Chroma 向量库 `similarity_search`，默认召回 k=50 篇文档
2. **重排序过滤**：DashScope `gte-rerank-v2` 对召回结果重排序，按阈值 0.5 过滤低质量结果，返回 top_n=10
3. **降级兜底**：重排序失败时降级为纯向量排序；无结果达标时兜底返回重排序最高分文档

### 2. 文档入库管道（前端驱动）

文档入库通过前端 **RAG 管理页面**（`/rag` 路由）完成，采用「预览 → 确认」两步流程：

**入库流程**：

1. **选择文件**（两种方式，互斥）：
   - **拖拽上传**：拖拽 `.md` 文件到上传区，或点击选择（multipart 直传）
   - **路径填写**：填写服务器上已有的 `.md` 文件路径（每行一个）
2. **预览分块**：点击「预览分块」→ 后端执行分块但不写入向量库，返回逐文件分块详情表格（序号、标题路径、内容预览、长度、切分类型）
3. **确认入库**：检查分块质量后点击「确认入库」→ 实际写入 Chroma 向量库

**分块策略**（两级）：
- 第一级：`ExperimentalMarkdownSyntaxTextSplitter` — 按 `#` / `##` / `###` 标题、代码块、水平线切分
- 第二级：`RecursiveCharacterTextSplitter` — chunk_size=1000，overlap=200（可配置）

**RAG 管理页面三大 Tab**：

| Tab | 功能 |
|-----|------|
| 文件处理 | 拖拽上传 / 路径填写 → 预览分块 → 确认入库 |
| 文档删除 | 按 ID 列表批量删除向量库文档 |
| 配置管理 | 在线编辑 `rag_config.yaml`（嵌入模型 / 分块器 / HNSW 索引参数），保存后运行时自动重载 |

**左侧健康面板**：实时展示集合名称、文档块数、嵌入模型、嵌入服务地址、持久化目录，支持每 10 秒自动刷新。

### 3. 长期记忆系统

独立于 RAG 文档库的记忆向量存储，提供 5 个工具：

| 工具 | 功能 |
|------|------|
| `save_memory` | 保存/更新结构化记忆（content、category、importance、metadata） |
| `search_memory` | 语义搜索记忆库（向量相似度 + 距离阈值过滤） |
| `delete_memory` | 按 key 删除记忆 |
| `get_memory` | 按 key 精确获取记忆 |
| `list_memory_keys` | 列出所有记忆 key（分页，最多 1000） |

记忆条目结构：`content`（自然语言）、`category`（分类）、`timestamp`（时间戳）、`importance`（重要性 0~1）、`metadata`（扩展元数据）。

### 4. 人机交互审批（HITL）

通过 `interrupt_on` 配置实现风险分级中断：

- `low`：工具调用完成后自动恢复执行
- `medium`：工具调用完成后中断，等待用户审批
- `high`：工具调用完成后中断，等待用户审批

用户可对中断的工具调用进行审批、编辑后批准或拒绝，通过 `/chat/{thread_id}/resume` 恢复执行。

### 5. 流式对话与工具调用

- **SSE 流式响应**：`text/event-stream`，前端处理 8 种事件类型
- **工具调用可视化**：工具调用卡片实时展示调用参数与结果
- **推理过程展示**：展示模型思维链
- **带文件附件**：支持上传 PDF / DOCX 后对话

### 6. Loop Engineering 条件驱动循环

基于 DeepAgents 框架自带的 `RubricMiddleware`，实现条件驱动的自主循环（Outer Loop）：

**工作机制**：

1. 用户设定 `rubric`（完成条件，"done 长什么样"）
2. Agent 正常执行（Inner Loop，ReAct 模式）
3. Agent 自然停止时，**独立评估器**（阿里云 ChatQwen，与执行 Agent 的 DeepSeek 形成对抗验证）判断 transcript 是否满足 rubric
4. 三态判断：
   - `satisfied` → 满足，终止
   - `needs_revision` → 注入反馈 + 自动循环继续（最多 10 轮）
   - `failed` → rubric 不可评估，终止
5. 达到 `max_iterations` 上限自动终止

**双层循环模型**：

```
Outer Loop（RubricMiddleware 编排层）
  目标条件 → 评估 → 反馈/终止
  
  Inner Loop（ReAct 执行层）
    思考 → 行动 → 观察 → ...
```

**对抗验证**：执行者（DeepSeek）与评估者（阿里云）使用不同模型、不同指令，评估者只读 transcript（防注入），不信任执行者的自我判断。

**使用方式**：
- 前端：点击输入框左下角 **Loop** 按钮，展开完成条件输入框
- API：请求体加 `rubric` 字段

```json
{
  "messages": [{"role": "user", "content": "帮我写一个快速排序并解释"}],
  "rubric": "回答必须包含：1. 可运行代码 2. 时间复杂度说明 3. 使用示例"
}
```

- 不传 `rubric` 时完全 no-op，向后兼容普通对话

**状态持久化**：评估状态（`_rubric_status` / `_rubric_iterations` / `_rubric_evaluations`）通过 `RubricState` 存入 SQLite 检查点，支持断点续跑。

### 7. 检查点分支导航

- 对话状态持久化到 SQLite 检查点
- 支持分支 / 重试 / fork
- 前端可视化检查点导航

### 8. 多模型切换与配置热更新

- 支持 DeepSeek / Ollama / 阿里云（DashScope）/ Moonshot 四种模型提供商
- 运行时通过 `ModelContext` 切换模型
- 配置热更新：每次初始化重新读取 `model_config.yaml`、`system_prompt.txt`、`mcp_server.json`

### 9. MCP 工具集成

通过 MCP 协议（fastmcp + langchain-mcp-adapters）接入外部工具服务器，Agent 启动时自动加载 MCP 工具集。

### 10. 文件管理

前端提供完整文件浏览器：

- 面包屑导航、目录树浏览
- 文件预览（Markdown / 文本 / 图片 / PDF）
- 创建 / 重命名 / 删除（含确认对话框）
- 拖拽上传、粘贴上传
- 面板可拖拽调整大小

### 11. CLI 命令行交互

除 Web 前端外，提供 `setup.bat` 菜单式 CLI（备选方式）：

- 交互式 Agent
- ChromaDB 可视化面板（Streamlit）
- 项目配置查看 / 管理
- Markdown 文档清洗
- 向量存储构建（文档入库，备选）
- ChromaDB 集合信息查看
- 待办事项管理

### 12. 微信机器人

通过 `backend/wechat_bot.py` 启动微信 AI Agent 机器人，可在微信中直接与知识库对话。

**核心特性**：

| 特性 | 说明 |
|------|------|
| 🔐 扫码登录 | 启动后生成二维码，微信扫码即可登录，凭据 24 小时内有效 |
| 💬 文本对话 | 接收微信文本消息，调用 LangGraph Agent（含 RAG 检索、长期记忆、技能管理全能力） |
| 📎 文件附件 | 支持在微信中发送 PDF / DOCX 文件，自动提取文本后交由 Agent 分析 |
| 🔗 文本-附件合并 | 先发文字再发文件时，3 秒内自动合并为一条消息处理 |
| 👤 会话隔离 | 每个微信用户独立 `thread_id`，对话上下文互不干扰 |
| 🔄 重置命令 | 发送「重置」即可清空当前会话，开启新对话 |
| 📊 可观测性 | 集成 Langfuse，支持全链路追踪，与 Web 端共享追踪数据 |
| 📝 日志系统 | 完整的日志记录，包括消息收发、文件处理、Agent 调用等 |

**启动方式**：

```powershell
# 先确保后端依赖已安装并激活虚拟环境
.venv\Scripts\activate
python backend/wechat_bot.py
```

启动后终端会显示二维码图片（自动打开）和扫码链接，用微信扫描即可开始对话。

    max_neighbors: 32

## 中间件配置

Agent 通过以下中间件链增强能力（`backend/core/assembled/middleware.py`）：

| 中间件 | 作用 |
|--------|------|
| `SummarizationToolMiddleware` | 手动摘要工具，750k tokens 触发自动摘要，保留 150k |
| `TruncateToolMessagesMiddleware` | 截断旧工具消息，保留最近 15 条 |
| `CodeInterpreterMiddleware` | QuickJS 代码解释器沙箱 |
| `RubricMiddleware` | Loop Engineering 条件驱动循环评估（deepseek评估器，max_iterations=10） |

框架通过 `create_deep_agent` 参数自动启用的中间件：

| 中间件 | 启用方式 | 作用 |
|--------|---------|------|
| `MemoryMiddleware` | `memory=["/AGENT.md"]` | 持久化记忆 |
| `SkillsMiddleware` | `skills=["/active_skills/"]` | 动态技能加载 |
| `SubAgentMiddleware` | 框架默认 | 通用子智能体 + 任务规划 |
